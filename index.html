Functions

Functions

×
A function is a process which takes some input, called arguments, and produces some output called a return value. Functions are really important in computing and math. They help us process data in useful ways.
Arity

Arity

×
The number of arguments a function takes. From words like unary, binary, ternary, etc. Addition, for example, takes two arguments, and so it is defined as a binary function or a function with an arity of two. A binary function must be given two and only two arguments.
Type Signatures

Type Signatures

×
Often functions in JavaScript will include comments that indicate the types of their arguments and return values. If a function accepts another function as an argument it is wrapped in parentheses. The letters a, b, c, d are used to signify that the argument can be of any type. Here we have a simple function, length, that accepts a word, of type String, and returns the count of characters in the string, which is a Number. The comment above the function is a signature line. It starts with the name of the function, then the separator "::" and then the actual description of the functions.
Purity / Side Effects

Purity / Side Effects

×
A function is pure if the return value is only determined by its input values, and does not produce side effects. Meaning, it can’t alter any external state. Pure functions are also extremely independent  —  easy to move around, refactor, and reorganize in your code, making your programs more flexible and adaptable to future changes. They are the simplest reusable building blocks of code in a program and makes the code easier to test.
Referential Transparency

Referential Transparency

×
When you call a function, it can be replaced with its resulting value without changing the behavior of the program. Basically, if a function consistently yields the same result for the same input, it is preferentially transparent.
Immutability

Immutability

×
When data is immutable, its state cannot change after it’s created. If you want to change an immutable object, you can’t. Instead, you create a new object with the new value. Strings and numbers are all immutable data.
Partial Application

Partial Application

×
Partial application can be described as taking a function that accepts some number of arguments, binding values to one or more of those arguments, and returning a new function that only accepts the remaining, un-bound arguments. Partial application helps create simpler functions from more complex ones by baking in data when you have it.
Currying

Currying

×
The process of converting a function that takes multiple arguments into a function that takes them one at a time. Each time the function is called it only accepts one argument and returns a function that takes one argument until all arguments are passed.
Higher Order Functions

Higher Order Functions

×
A higher-order function can take a function as an argument or return one as a result. You should always remember that functions are values, meaning you can pass them around, much like you can do with variables.
Functor

Functor

×
A Functor is a container that can be treated with a special function(say fmap or map) which applies any unary function to every content of this container and returns a container with the resulting values as its contents. Map function in arrays takes an array and applies a particular function to all of its elements one by one and returns another array.
Applicative Functor

Applicative Functor

×
An applicative functor is an object with an ap function. ap applies a function in the object to a value in another object of the same type.
Monad

Monad

×
A monad is an object with "return" and "flatmap" functions. Flatmap is like map except it un-nests the resulting nested object. A monad is based on a simple symmetry — A way to wrap a value into a context, and a way to unwrap the value from the context.
